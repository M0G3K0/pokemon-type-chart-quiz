## 📋 概要

すべてのUIコンポーネントについて、各バリアント・ステートの使い分けガイドラインを明確に文書化する。

## 🎯 目的

開発者（AI含む）が適切なコンポーネントとバリアントを選択できるよう、「どの場面でどれを使うべきか」を明示することで：
- 一貫性のあるUI設計
- 意図しないスタイルの誤用防止
- 新規開発時の迷い軽減

## ⬜ 対応内容

### 対象コンポーネント

- [ ] **Button**: Primary / Secondary / Ghost / Danger の使い分け
- [ ] **RadioButton**: 通常選択 / Quiz用フィードバックステートの使い分け
- [ ] **Chip / TypeChip**: 情報表示 vs インタラクティブ用途
- [ ] **Card / Surface**: 責務分離と使い分け
- [ ] **Badge**: 結果表示（win/lose/neutral）の使い方
- [ ] **Avatar**: 形状バリアント（circle/rounded）の使い分け
- [ ] **Heading**: size vs level の違い

### 追加場所（案）

1. **NgDoc Styleタブ**: 各コンポーネントのStyleタブにユースケース説明を追加
2. **docs/guidelines/component-usage.md**: 全体的なガイドライン文書

### 例：Button バリアント使い分け

| バリアント | 用途 | 例 |
|---|---|---|
| **Primary** | 主要アクション、CTA | 「スタート」「送信」 |
| **Secondary** | 副次的アクション、アウトラインスタイル | 「キャンセル」「戻る」 |
| **Ghost** | 最小限のスタイル、インラインアクション | カード内リンク、ナビゲーション |
| **Danger** | 破壊的・元に戻せないアクション | 「削除」「リセット」 |
