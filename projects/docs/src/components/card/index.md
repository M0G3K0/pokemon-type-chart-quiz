> 関連するコンテンツをグループ化するコンテナコンポーネント

## Overview

`pt-card`は、関連するコンテンツをグループ化して表示するためのMoleculeレベルのコンポーネントです。ヘッダー、コンテンツ、フッターの構造を持ち、一貫したスタイリングを提供します。

**複合コンポーネント**:
- `pt-card`: カード全体のコンテナ
- `pt-card-header`: ヘッダーエリア（タイトル、アクション）
- `pt-card-content`: メインコンテンツエリア
- `pt-card-footer`: フッターエリア（アクションボタン等）

## When to use ✅

| シチュエーション | 説明 |
|------------------|------|
| **関連情報のグループ化** | プロフィール、商品情報など |
| **アクション可能なコンテンツ** | フッターにボタンを配置 |
| **視覚的な区切り** | 他のコンテンツと明確に分離 |
| **階層構造の表現** | 浮いた印象で重要度を示す |

## When not to use ❌ (Anti-patterns)

| アンチパターン | 理由 | 代替案 |
|----------------|------|--------|
| **単純な背景ボックス** | Cardは構造を持つ | `pt-surface` |
| **カード内にカード** | 複雑さが増す | フラットな構造に再設計 |
| **1つのみの内部要素** | Cardの構造が不要 | `pt-surface` |

## Elevation 選択ガイド

> 参照: [Surface/Card 責務分離](../../decisions/surface-card-responsibility.md)

### Flat

**使用する場面:**
- **影が不要な場合**
- 背景色のみで区別できる場合
- 密なUIでカードが多数並ぶ場合

**使用しない場面:**
- カードが重要な情報を持つ場合（目立たない）

```html
<pt-card elevation="flat">
  <!-- 境界線のみ、影なし -->
</pt-card>
```

### Raised (デフォルト)

**使用する場面:**
- **標準的なカード表示**
- 適度に浮いた印象が必要
- ほとんどの場面で推奨

**使用しない場面:**
- 密なレイアウト（影が重なって見づらい）

```html
<pt-card elevation="raised">
  <!-- デフォルト、やや浮いた印象 -->
</pt-card>
```

### Overlay

**使用する場面:**
- **モーダル、ドロップダウン、ポップオーバー**
- コンテンツの上に重なって表示される場合
- 強い視覚的分離が必要

**使用しない場面:**
- 通常のページ内カード（影が強すぎる）

```html
<pt-card elevation="overlay">
  <!-- モーダル内など、最前面に表示 -->
</pt-card>
```

## Size 選択ガイド

| サイズ | パディング | 用途 |
|--------|------------|------|
| `sm` | 8px | コンパクト、リスト内カード |
| `md` | 16px | 標準（デフォルト） |
| `lg` | 24px | ゆったり、ヒーローカード |

---

## pt-surface との使い分け

| 判断基準 | 選択 |
|----------|------|
| Header/Content/Footer の構造が必要 | `pt-card` |
| 単純な背景ボックスで十分 | `pt-surface` |
| デフォルトで影が欲しい | `pt-card` |
| 影なしで柔軟にカスタマイズ | `pt-surface` |

---

## Accessibility

- カード自体にはARIA属性不要
- 内部のインタラクティブ要素がアクセシブルであること
- フォーカス順序: ヘッダー → コンテンツ → フッター

## Related Components

- `pt-surface`: 単純なコンテナスタイリング
- `pt-button`: カードフッターのアクションボタン
- `pt-heading`: カードヘッダーのタイトル
