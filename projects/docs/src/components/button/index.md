> アクションの実行を促すボタンコンポーネント

## Overview

`pt-button`は、ユーザーのアクションを促すためのコンポーネントです。一貫したスタイリングと、variant/sizeによる柔軟なカスタマイズをサポートします。

**主な特徴**:
- **4種類のvariant**: primary, secondary, ghost, danger
- **3種類のサイズ**: sm, md, lg
- **disabled状態**: 操作不可の視覚的表現

## When to use ✅

| シチュエーション | 説明 |
|------------------|------|
| **主要なアクション** | フォーム送信、確定など（primaryを使用） |
| **補助的なアクション** | キャンセル、戻るなど（secondaryを使用） |
| **破壊的なアクション** | 削除、リセットなど（dangerを使用） |
| **控えめなアクション** | リンク的な振る舞い（ghostを使用） |

## When not to use ❌ (Anti-patterns)

| アンチパターン | 理由 | 代替案 |
|----------------|------|--------|
| **1画面にPrimaryを複数配置** | 視覚的な優先度が曖昧になる | 主要アクションは1つに絞る |
| **曖昧なラベル** | 「ここをクリック」などはUXが悪い | 動詞で始まる明確なラベル |
| **リンクの代わり** | ナビゲーションはリンクの責務 | `<a>`タグまたは将来の`pt-link` |

## Variant 選択ガイド

### Primary

**使用する場面:**
- ページ/セクションの**最も重要なアクション**
- ユーザーに最も実行してほしい操作
- フォームの送信ボタン

**使用しない場面:**
- 1画面に複数配置（優先度が曖昧になる）
- ユーザーを驚かせる可能性のあるアクション

```html
<!-- ✅ 良い例: ページに1つの主要アクション -->
<pt-button variant="primary">クイズを始める</pt-button>

<!-- ❌ 悪い例: 複数のPrimary -->
<pt-button variant="primary">保存</pt-button>
<pt-button variant="primary">別名で保存</pt-button>
```

### Secondary

**使用する場面:**
- Primaryと並べる**副次的なアクション**（キャンセル、戻るなど）
- 強調は必要だが最優先ではない操作
- 複数の同等レベルの選択肢

**使用しない場面:**
- ナビゲーションリンク（ghostを使用）
- 最も重要なアクション（primaryを使用）

```html
<!-- ✅ 良い例: Primary + Secondary のペア -->
<pt-button variant="primary">保存</pt-button>
<pt-button variant="secondary">キャンセル</pt-button>
```

### Ghost

**使用する場面:**
- **最小限の視覚的強調**で十分な場合
- ナビゲーション、ツールバー内のアクション
- テキストリンクに近い動作だがボタン領域が必要な場合

**使用しない場面:**
- 重要なアクション（目立たなすぎる）
- フォームの送信ボタン

```html
<!-- ✅ 良い例: ツールバー内 -->
<pt-button variant="ghost">詳細を見る</pt-button>
```

### Danger

**使用する場面:**
- **破壊的・取り消し不可能なアクション**
- 削除、リセット、退会など
- 確認ダイアログ内での危険な選択肢

**使用しない場面:**
- 通常のアクション（赤色は警告の意味が強い）
- 頻繁に使用するボタン

```html
<!-- ✅ 良い例: 削除確認ダイアログ内 -->
<pt-button variant="danger">削除する</pt-button>
<pt-button variant="secondary">キャンセル</pt-button>
```

## Size 選択ガイド

| サイズ | 用途 |
|--------|------|
| `sm` | 密なUI、テーブル行内、ツールバー、カード内の副次的アクション |
| `md` | **標準（デフォルト）**。ほとんどの場面で使用 |
| `lg` | **CTA（Call To Action）**。ユーザーに最も実行してほしいアクション |

### 具体的なシナリオ

| シナリオ | 推奨サイズ | 理由 |
|----------|-----------|------|
| ランディングページの主要アクション | `lg` | 画面の主役（CTA） |
| フォーム送信ボタン | `md` | 標準的なアクション |
| カード内のアクションボタン | `md` | 標準的なアクション |
| モーダルの「確認」「キャンセル」 | `md` | 標準的なアクション |
| テーブル行内の「編集」「削除」 | `sm` | 密なUI |
| ツールバー内のアクション | `sm` | コンパクトなUI |

### CTA（Call To Action）とは？

画面内で**ユーザーに最も実行してほしいアクション**のこと。

```html
<!-- ✅ CTA: 画面の主役となるボタン -->
<pt-button variant="primary" size="lg">今すぐ始める</pt-button>

<!-- ✅ 通常アクション: フォーム送信など -->
<pt-button variant="primary" size="md">送信</pt-button>

<!-- ✅ 副次的アクション: 補足的な操作 -->
<pt-button variant="ghost" size="sm">詳細を見る</pt-button>
```

## Accessibility

- `disabled`属性が適用された場合、クリックイベントは発火しません
- キーボード操作: `Tab`でフォーカス、`Space`/`Enter`でクリック
- フォーカスリング表示（`:focus-visible`）

## Related Components

- `pt-card`: カード内のアクションボタンとして使用
- `pt-radio-button`: 選択肢としてのボタン的UI
